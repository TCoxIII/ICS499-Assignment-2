### AI Effectiveness Analysis

## Topic Discussions

### What did AI do well?

AI gave very good insight and digestible explanations of CRM, including the commercial and open-source CRM companies. Through proper prompting, AI helped explain the assignment topics clearly and helped create a plan that addressed everything required.

### What did AI struggle with?

AI struggled with where each subtask should go within the repository and how to format some tables properly for GitHub. Some tables were not as human-readable as expected because the columns did not always align cleanly. I probably could have improved this by iterating more with the AI and asking for better Markdown formatting.

### What information needed to be validated?

Pretty much all information needed to be validated to make sure accurate and relevant information was being used in the assignment. This was especially important for CRM product features, pricing, open-source project details, and official documentation because these details can change over time.

### What surprised you?

The planning phase was very straightforward. After providing the assignment link and asking for a plan, AI was able to break the assignment into clear parts and subtasks. Then, by using more specific prompts, I was able to get better output for each section of the assignment.

### What would you do differently next time?

Next time, I might use a CLI-based AI tool that can interact directly with the project files. This would reduce the amount of copy and paste needed after getting the desired output. It would also make it easier to update files directly inside the repository.

### Would you trust AI for software research?

Yes, I would use AI for software research. I currently use AI in my job, and it is helpful for understanding concepts, organizing tasks, reviewing ideas, and generating starting points. However, I would still review and validate the information before using it in a final assignment or professional setting.
